When you want to install a new program on Ubuntu, but it's not in apt . . .

Published: Thu 18 December 2014
By Fulton Byrne

In When You Want . . ..

I wanted to install Intellij today to build an Android App I was thinking of. Just for kicks, because I'm sure something like my idea exists out there, but hey I'm looking for jobs, might as well practice Java.

I used Eclipse in school, but I really didn't enjoy how slow it was. So, I wanted to try out Intellij. Doink! It's not in a PPA (Personal Package Archives). Well, I said to myself, today is the day I grow up and start doing things "right".

Step 1: Download and Unpack

Went to the website, downloaded it. First obstacle check, second obstacle how do I unpack thee?

tar -xzf download.tar.gz

Break it down:

  • tar an archiving/unarchiving utility.
  • -x extract that package
  • -z put it through gzip because it has a .gz ending.
  • -f output it to a file? I'm not really sure about this guy, but it works!

Step 2: Move that Folder Somewhere not Annoying

I decided /usr/local was a good spot, because it was close to /usr/local/bin, and there were some other utilities like heroku sitting in there as well.

Step 3: Access that Program from Anywhere (Like You Know What You're All About).

So this was my first time making a Symbolic Link, which is like a Windows Shortcut, but I was rather well pleased with the results, because I didn't have to further clutter my $PATH variable.

Since /usr/local/bin is already on our path, anything I throw in there is accessible from the command line; no matter which directory I'm in. IntelliJ has a bash script in it's /bin folder that you use to run the program, so I decided to put a symbolic link to that script in /usr/local/bin using the ln command.

ln -s /source/path/to/file /dest/path/to/link

Et voila, symbolic link, and I can just type intellij or whatever I called the link file, and bam Intellij. I could have even called it stupidj and it would still work, but it's not stupid so I'll call it intellij. Oh yeah, and the -s makes it a symbolic link, rather than a hard link. This just means we can swap out that file for newer versions, but if we still keep the same name our link still points to it.
